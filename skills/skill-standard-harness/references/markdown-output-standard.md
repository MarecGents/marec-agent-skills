# Markdown 输出规范

本规范定义了技能输出文档的通用格式要求，确保跨技能文档风格统一、结构清晰。

---

## 标题层级规则

- 使用清晰的 Markdown 层级标题：`#` → `##` → `###` → `####`
- `#`（一级标题）仅用于文档标题，每文件限一次
- `##`（二级标题）用于主要章节
- `###`（三级标题）用于章节下的子节
- 不要跳级使用（如 `#` 后直接 `###`）

## 时间戳格式

所有日志记录中的时间戳采用统一格式：

```
YYYY-MM-DD HH:mm
```

示例：`2026-06-27 15:30`

## 技术术语处理

- 代码、标识符、文件路径、shell 命令和未翻译的技术术语**保持原文**，不翻译
- 例如：`Git`、`npm`、`README.md`、`planning-with-files`、`MCP`、`CLI` 等不翻译
- 版本号统一格式：`vX.Y.Z`（如 `v1.13.0`）

## 表格格式

- 对比表、参数说明表等使用标准 Markdown 表格语法
- 表格第一行为表头，第二行为对齐分隔线
- 内容较长时使用合理的列宽，必要时折行

```markdown
| 参数 | 说明 |
|------|------|
| `--output` | 输出路径 |
| `--verbose` | 详细输出模式 |
```

## 最终日志文档必备要素

一份完整的任务日志文档应包含以下内容：

1. **任务概述** — 名称、目标、范围
2. **执行过程摘要** — 每个阶段做了什么
3. **关键发现与决策** — 过程中产生的洞察和选择
4. **遇到的问题与解决方案** — 异常场景的处理记录
5. **最终结果与状态** — 任务完成情况、产出物列表

## 文件命名规范

- 使用 `{概括性名称}-{YYYY-MM-DD-HHmm}.md` 格式
- 名称部分概括文档内容，使用中文或英文连字符
- 时间部分精确到分钟

示例：`用户登录模块重构-2026-06-27-1530.md`、`replication-log-2026-06-27.md`

## 代码块使用

- 命令行示例使用 `bash` 标记语言
- 配置文件使用对应的语言标记（`json`、`yaml`、`toml` 等）
- 文件路径和变量用反引号包裹：`` `path/to/file` ``

## 列表使用

- 有序列表（`1.`）用于步骤说明和优先级排序
- 无序列表（`-`）用于要点列举和属性描述
- 嵌套列表缩进 2 个空格
